Слои системы

СлойТехнологияРольКлючевые объекты
OLAP Databricks Аналитическое хранилище. Данные грузит BI-слой (Д. Короткова), Кабинет читает read-only. dnipro_m.raw_amazon_fba_inventory_ledger_summary
ETL Databricks Notebook Перенос и нормализация: снапшот остатков + генерация инцидентов по правилам. Kabinet — Stock Loader · ячейки 1–3
OLTP Lakebase (Postgres) Операционное состояние системы: остатки, журнал инцидентов, черновики заказов. stock_local incidents sales_velocity draft_orders +4
UI Streamlit Cloud Приложение для операций: аналитика, журнал задач, прогноз. Подключение через OAuth M2M. kabinet-dashboard-dnipro-m.streamlit.app

Контур инцидентов

stock_local → правила (остаток = 0 · остаток ≤ 3) → incidents → страница «Инциденты» → действие человека

Система сама находит проблемы при каждом обновлении данных и ведёт их жизненный цикл: открытие без дублей, автозакрытие при восстановлении стока. Отчёты отвечают на вопрос «почему так», журнал инцидентов — «что делать сегодня».
